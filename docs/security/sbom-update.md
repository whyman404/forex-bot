## 1. Why This Document

Phase 3a adds two new community libraries that are now critical-path for live trading:
- `tradingview-mcp-server` — MCP server bridging our engine to TV scanner.
- `tradingview-ta` — Python lib that calls TV scanner endpoint.

Both are community-maintained. **Supply chain attestation matters more here than for vendored official libs.** This doc records pinned versions, provenance, security contacts, and audit dates.

### 2.3 Transitive Dependencies of Concern

Both libs pull common transitive deps. Watch closely:
- `requests` (Python) — TLS, redirect handling.
- `pandas` (Python — may be optional) — heavy dep; CVE-prone.
- `pydantic` (Python) — we already use; share version.
- `urllib3` (Python) — historical CVE pattern.
- `axios` / `node-fetch` (Node, if MCP server is Node) — TLS, SSRF risk.
- `ws` (Node) — websocket DoS history.

Full transitive list: see CycloneDX SBOM artifact in `dist/sbom/` per build.

## 3. Supply Chain Controls

### 3.1 At install
- `pip install --require-hashes` (Python) — hashes pinned in lockfile.
- `npm ci` (Node) — exact lockfile install.
- No `pip install --pre` / no `npm install --tag`.
- No installation from a fork or branch.

### 3.2 At build
- `pip-audit` on lockfile → fail on Critical / High.
- `npm audit --audit-level=high` → fail.
- `trivy fs` on built image → fail on Critical / High.
- `cyclonedx-py` / `cyclonedx-node` → produce SBOM artifact, archive 5 years.
- Sigstore signing if available (`cosign sign`); Rekor entry recorded.

### 3.3 At runtime
- Process isolation — TV adapter runs as a separate worker (`trading-engine-tv-worker`) with:
  - **No access** to `broker_credentials` DB rows (role-level grant).
  - **No access** to KEK env var.
  - **Network egress allowlist**: only `scanner.tradingview.com` + `api.tradingview.com` + our internal API.
  - **Read-only filesystem** except `/tmp` and log dir.
  - **Non-root** user.
- Resource limits — cgroup memory + CPU caps; OOM-restart automatic.

### 3.4 Monitoring
- Renovate / Dependabot → weekly PR.
- Security label on PR → human review required (NEVER auto-merge supply-chain).
- Upstream watch: GitHub Releases subscription for both repos → Slack alert.
- PyPI / npm advisory database subscribed.
- CVE feeds: NVD + GitHub Advisory Database.

### 3.5 Incident response trigger
- New CVE published on either lib → triage within 24h.
- Critical CVE → patch within 7 days (Phase 2 SLA).
- If exploit-in-the-wild on either lib → halt `tv_signal` strategies + investigate within 4h.

## 4. Secrets Note

Neither library uses an API key currently — `tradingview-ta` calls the public scanner endpoint without authentication. **No secret to manage today.** If we adopt **paid TradingView API** in the future:
- Add `TRADINGVIEW_API_KEY` to `secrets-audit.md` Section 1 (item 18).
- Apply envelope encryption pattern (same as MT5 broker credentials).
- Restricted scope: read-only on scanner endpoints.
- Rotation: quarterly + on incident.
- Blast radius: low-medium (read-only data; rate-limit abuse mainly).
